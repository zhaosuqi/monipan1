# 实时模拟盘Web监控使用指南

## 功能特性

### ✨ 已实现

1. **实时K线显示**
   - 使用 Lightweight Charts 图表库
   - 预加载历史K线数据（1000条）
   - 实时推送新K线数据
   - 自动更新图表

2. **系统日志**
   - 实时输出系统日志
   - 保持最新50条日志
   - 日志级别标识（INFO/WARNING/ERROR）

3. **状态监控**
   - 连接状态指示
   - 交易对信息
   - K线数量统计
   - 最新K线时间

4. **最新K线信息**
   - 开盘价
   - 最高价
   - 最低价
   - 收盘价

## 🚀 快速开始

### 步骤1: 配置环境

确保已切换到测试网模式并配置API密钥：

```bash
# 如果还没配置
./switch_to_testnet.sh

# 编辑.env文件，填入API密钥
vim .env
```

### 步骤2: 启动服务

```bash
./start_live_web.sh
```

服务将在 `http://localhost:5001` 启动

### 步骤3: 访问监控页面

在浏览器中打开: http://localhost:5001

## 🔧 工作原理

### 后端架构

```python
# 1. 启动时
- 连接币安测试网API
- 预加载1000条历史K线
- 启动后台线程获取实时K线
- 启动WebSocket服务

# 2. 运行时 (每分钟)
- 后台线程获取最新K线
- 推送到前端 via WebSocket
- TradeEngine处理交易信号
- 输出日志

# 3. 前端
- 连接WebSocket
- 接收历史K线并绘制
- 接收新K线并更新
- 显示日志和状态
```

### WebSocket事件

| 事件 | 方向 | 说明 |
|------|------|------|
| `connect` | 客户端→服务器 | 客户端连接 |
| `initial_klines` | 服务器→客户端 | 发送历史K线 |
| `new_kline` | 服务器→客户端 | 推送新K线 |
| `log_update` | 服务器→客户端 | 推送日志 |
| `status_update` | 服务器→客户端 | 状态更新 |

## ⚙️ 配置说明

### 端口配置

修改 `live_web_app.py` 的端口：
```python
PORT = 5001  # 可以修改为其他端口
socketio.run(
    app,
    host='0.0.0.0',
    port=PORT,
    debug=False
)
```

### K线数量

修改预加载的K线数量：
```python
# 在 api_historical_klines 路由
limit = request.args.get('limit', 1000, type=int)

# 在 main 函数
historical_klines = get_historical_klines(limit=1000)
```

### 刷新频率

修改实时K线获取频率：
```python
# 在 live_kline_thread 函数
time.sleep(60)  # 秒数，默认60秒
```

## 🐛 故障排查

### 页面无法访问

1. 检查服务是否启动
2. 检查端口是否被占用: `lsof -i :5001`
3. 查看后端日志输出

### K线不更新

1. 检查API密钥是否正确
2. 检查网络连接
3. 查看浏览器控制台错误
4. 查看后端日志

### WebSocket连接失败

1. 检查防火墙设置
2. 查看浏览器控制台
3. 确认服务正常运行

## 🎯 与回测模式的区别

| 特性 | 回测Web | 实时Web |
|------|---------|---------|
| 数据源 | 本地数据库 | 币安API |
| 更新方式 | 手动刷新 | 实时推送 |
| 图表更新 | 静态 | 动态 |
| 用途 | 查看历史回测 | 监控实时交易 |

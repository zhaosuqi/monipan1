# 预热天数更新说明

## 更新内容

将回测预热天数从 **50天** 增加到 **200天**,以支持1天(1d)MACD指标的计算。

## 原因

### 指标周期需求
- **1分钟MACD**: 需要约26根1分钟K线 (EMA26)
- **1小时MACD**: 需要约26根1小时K线 = 26×60 = 1,560根1分钟K线
- **4小时MACD**: 需要约26根4小时K线 = 26×240 = 6,240根1分钟K线
- **1天MACD**: 需要约26根1天K线 = 26×1440 = 37,440根1分钟K线

### 安全余量
考虑到:
- 指标计算的稳定性
- 跨周期采样的数据对齐
- 边界情况的处理

**建议至少200天 (288,000根1分钟K线) 的预热数据**

## 配置更新

### 环境变量
```bash
# .env 文件
REPLAY_WARM_DAYS=200  # 从50更新为200
```

### Python代码
```python
# core/config.py
self.REPLAY_WARM_DAYS = int(os.getenv('REPLAY_WARM_DAYS', '200'))
```

### 测试脚本
```python
# simple_replay.py
warm_days = 200  # 从50更新为200
preload_start = "2023-06-15T00:00:00"  # 前200天
```

## 数据量对比

| 预热天数 | K线数量 | 数据大小 |
|---------|--------|---------|
| 50天    | 72,000 | ~5 MB   |
| 200天   | 288,000 | ~20 MB  |

## 回测示例

### 短期回测 (3天)
```
预热数据: 2023-06-15 至 2023-12-31 (200天, 288,000条)
回测数据: 2024-01-01 至 2024-01-03 (3天, 4,320条)
总计: 292,320条K线
执行时间: < 1秒
```

### 月度回测 (30天)
```
预热数据: 2023-06-15 至 2023-12-31 (200天, 288,000条)
回测数据: 2024-01-01 至 2024-01-30 (30天, 43,200条)
总计: 331,200条K线
执行时间: ~5秒
```

### 年度回测 (365天)
```
预热数据: 2023-06-15 至 2023-12-31 (200天, 288,000条)
回测数据: 2024-01-01 至 2024-12-31 (365天, 525,600条)
总计: 813,600条K线
执行时间: ~30秒
```

## 影响范围

### 受益的指标
✅ 1分钟MACD (macd15m)
✅ 15分钟MACD (可通过1分钟数据计算)
✅ 1小时MACD (macd1h)
✅ 4小时MACD (macd4h)
✅ **1天MACD (macd1d)** ⭐ 新增支持
✅ KDJ指标 (所有周期)

### 性能影响
- **内存占用**: 略增 (约20MB预热数据)
- **加载时间**: 略增 (约0.5秒)
- **计算准确性**: 大幅提升 (特别是1天MACD)

# 币安API数据缺口问题说明

## 问题描述

在补充历史数据时发现，从数据库最后一条记录（2025-12-19 23:59:00）继续获取数据时，币安API返回的数据从 2025-12-20 07:20:00 开始，中间缺失了 7 小时 20 分钟的数据。

## 数据验证

### 数据库统计

```sql
-- 2025-12-19 的数据
SELECT MIN(open_time), MAX(open_time), COUNT(*)
FROM klines_1m
WHERE open_time >= '2025-12-19' AND open_time < '2025-12-20'

结果: 2025-12-19T00:00:00 | 2025-12-19T23:59:00 | 1440 条
✅ 数据完整（一整天 1440 分钟）
```

```sql
-- 2025-12-20 的数据
SELECT MIN(open_time), MAX(open_time), COUNT(*)
FROM klines_1m
WHERE open_time >= '2025-12-20' AND open_time < '2025-12-21'

结果: 2025-12-20T07:20:00 | 2025-12-20T23:59:00 | 1000 条
❌ 数据不完整，缺失前 440 条（00:00:00 ~ 07:19:00）
```

## 问题分析

### 请求日志

```
数据库最后K线open_time: 2025-12-19 23:59:00+00:00
请求startTime: 1766188800000 (2025-12-20 00:00:00+00:00)
请求endTime: 1766275140000 (2025-12-20 23:59:00+00:00)
API返回: 2025-12-20 07:20:00+00:00 ~ 2025-12-20 23:59:59.999000+00:00
```

### 可能的原因

1. **币安API数据限制**
   - 币安实盘API可能不保留完整的历史数据
   - 较早的历史数据可能已被清理或归档
   - API只提供最近一段时间的数据

2. **数据维护窗口**
   - 2025-12-20 00:00:00 ~ 07:20:00 可能是币安的维护时间
   - 这段时间市场可能暂停交易

3. **API行为**
   - 当请求的时间点数据不可用时，币安API返回它能提供的最老数据
   - 而不是返回错误或空结果

## 币安API限制说明

### 数据保留期限

根据币安API文档和实际使用经验：

| K线周期 | 数据保留期限 |
|---------|------------|
| 1分钟 | 最近几天到几周 |
| 5分钟 | 最近几个月 |
| 1小时 | 更长时间 |
| 1天 | 可能永久保留 |

**注意**: 具体保留期限币安未明确说明，可能随时变化。

### API行为

- 当 `startTime` 过早时，API返回它能提供的最老数据
- 不会返回错误或警告
- 需要调用方自行判断返回数据是否符合预期
